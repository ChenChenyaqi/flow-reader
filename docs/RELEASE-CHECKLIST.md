## 🚀 发布流程

### 1. 准备 Google 账户

- [ ] 注册 Google 账户
- [ ] 注册 Chrome Web Store 开发者
  - [ ] 访问：https://chrome.google.com/webstore/devconsole
  - [ ] 支付 $5 美元注册费
  - [ ] 验证邮箱

### 2. 创建项目

- [ ] 登录 [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
- [ ] 点击"新建项目"
- [ ] 填写项目信息：
  - [ ] **扩展名称**: FlowReader
  - [ ] **描述**: AI 驱动的英语学习助手...
  - [ ] **类别**: 教育类
  - ] ] **语言**: 中文, English

### 3. 上传扩展

- [ ] 填写"商店列表"
  - [ ] 上传扩展包 .zip
  - [ ] 上传图标
  - [ ] 上传截图
  - [ ] 填写描述（从 `docs/STORE-LISTING.md` 复制）

### 4. 隐私与权限

- [ ] 添加隐私政策链接（`docs/PRIVACY-POLICY.md`）
  - [ ] 上传到 GitHub Pages 或其他可访问位置
  - [ ] 在商店列表中填写链接
- [ ] 检查权限声明
  - [ ] `storage` - 用于存储配置
  - [ ] `activeTab` - 用于获取当前页面内容
  - [ ] `scripting` - 用于注入内容脚本

### 5. 定价与分发

- [ ] **定价**: 免费
- [ ] **分发区域**: 全球
- [ ] **可见性**: 公开

### 6. 提交审核

- [ ] 检查所有信息
- [ ] 确认符合 Chrome Web Store 政策
- [ ] 点击"提交审核"

## ⏳ 等待审核

### 审核周期

- **通常**: 1-3 个工作日
- **可能更长**: 首次提交或功能较复杂

### 审核期间

- [ ] 查看审核状态（开发者后台）
- [ ] 如需补充材料，及时响应
- [ ] 保持邮箱畅通，接收审核通知

### 常见被拒原因及应对

| 被拒原因 | 解决方案 |
|---------|---------|
| 功能描述不准确 | 修改描述，确保真实准确 |
| 权限声明过多 | 只申请必要的权限 |
| 隐私政策缺失 | 添加隐私政策链接 |
| 界面不符合规范 | 按照 Material Design 调整 |
| 包含恶意代码 | 检查代码，确保无恶意行为 |
| 图标/截图不规范 | 按照规范重新制作 |
